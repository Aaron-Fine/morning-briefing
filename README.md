# Morning Digest

A self-hosted daily briefing email generated by AI, tailored to your interests. Runs as a Docker container on Unraid (or anywhere Docker runs).

## What It Does

Every morning at 6:00 AM MT, this container:

1. **Collects** data from multiple sources:
   - YouTube (yt-dlp вҖ” new uploads from your Always Watch channels, no API key)
   - RSS feeds (Kagi, CNN, NPR, Axios, Al Jazeera, The Atlantic, Breaking Defense, SpaceNews, etc.)
   - Local news RSS (Cache Valley Daily, Herald Journal)
   - Open-Meteo weather (Cache Valley forecast, no key needed)
   - Finnhub market quotes (SPY, DIA, XAR, XLE)
   - Launch Library 2 (upcoming space launches, no key needed)
   - Come Follow Me lesson schedule (LDS)

2. **Sends** all collected data to Kimi K2.5 via the Fireworks AI API with a detailed editorial prompt encoding your preferences

3. **Renders** the output into a polished HTML email

4. **Delivers** the email to your Gmail via SMTP

## Quick Start

### 1. Get your API keys

| Key | Where | Free? |
|-----|-------|-------|
| Fireworks AI API | [fireworks.ai/account/api-keys](https://fireworks.ai/account/api-keys) | Pay-per-use (~$0.05вҖ“0.15/day) |
| Finnhub API | [finnhub.io](https://finnhub.io) | Free tier |
| Gmail App Password | [myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords) | Yes |

### 2. Configure

```bash
cp .env.example .env
# Edit .env with your API keys

# Edit config.yaml:
# - Set delivery.to_address to your Gmail
# - Adjust topics, channels, feeds, and schedule as desired
```

### 3. Test locally

```bash
# Collect all sources and dump to output/sources.json (no AI call)
docker compose run --rm morning-digest python digest.py --sources-only

# Full run: collect + AI synthesis + render HTML, saved to output/last_digest.html (no email)
docker compose run --rm morning-digest python digest.py --dry-run

# Production: collect + synthesize + send email
docker compose up -d
```

### 4. Deploy to Unraid

See the [Unraid section](#unraid) below.

## Unraid

### Option A: Compose Manager plugin (recommended)

This is the cleanest approach if you have the [Compose Manager](https://forums.unraid.net/topic/114415-plugin-docker-compose-manager/) plugin installed.

**1. Copy the project to appdata:**
```bash
mkdir -p /mnt/user/appdata/morning-digest
cp -r . /mnt/user/appdata/morning-digest/
```

**2. Create your `.env` file:**
```bash
cd /mnt/user/appdata/morning-digest
cp .env.example .env
nano .env   # fill in your keys
```

**3. Edit `config.yaml`** вҖ” set your `delivery.to_address` and adjust preferences.

**4. In the Unraid UI:**
- Go to **Docker вҶ’ Compose** tab
- Click **Add Stack**
- Name it `morning-digest`
- Point the compose file path to `/mnt/user/appdata/morning-digest/docker-compose.yml`
- Click **Up** to start

**5. Test it:**

Open an Unraid terminal and run:
```bash
cd /mnt/user/appdata/morning-digest
docker compose run --rm morning-digest python digest.py --dry-run
```

Then check `output/last_digest.html` вҖ” copy the path and open it in a browser via Unraid's file manager, or `scp` it to your machine.

---

### Option B: Unraid Docker UI (manual)

If you don't use the Compose plugin, you'll need to pre-build the image.

**1. Build the image on your machine (or on Unraid via terminal):**
```bash
cd /mnt/user/appdata/morning-digest
docker build -t morning-digest:latest .
```

**2. In the Unraid Docker tab вҶ’ Add Container:**

| Field | Value |
|-------|-------|
| Name | `morning-digest` |
| Repository | `morning-digest:latest` |
| Network Type | Bridge |
| Restart | `unless-stopped` |

**3. Add these Environment Variables:**

| Variable | Value |
|----------|-------|
| `TZ` | `America/Denver` |
| `FIREWORKS_API_KEY` | your Fireworks key |
| `FINNHUB_API_KEY` | your Finnhub key |
| `SMTP_USER` | your Gmail address |
| `SMTP_PASSWORD` | your Gmail App Password |

**4. Add these Path mappings:**

| Container path | Host path | Access |
|---------------|-----------|--------|
| `/app/config.yaml` | `/mnt/user/appdata/morning-digest/config.yaml` | Read Only |
| `/app/output` | `/mnt/user/appdata/morning-digest/output` | Read/Write |

**5. Click Apply.** The container will start and wait until 6:00 AM MT to send the first digest.

## Configuration Reference

### config.yaml

**`schedule.cron`** вҖ” When to send. Default: `0 6 * * *` (6:00 AM daily)

**`delivery.to_address`** вҖ” Your Gmail address (required)

**`llm.model`** вҖ” AI model. Default: `accounts/fireworks/models/kimi-k2p5`

**`llm.temperature`** вҖ” 0.3 keeps output consistent; raise toward 0.6 for more creative prose

**`topics.primary / secondary / tertiary`** вҖ” Topic priority tiers. Primary always gets coverage; tertiary only when something significant happens

**`youtube.always_watch`** вҖ” Your channels. Uses `handle` (the `@username` on YouTube). Every new upload within `lookback_hours` appears in the digest

**`rss.feeds`** вҖ” Feed list with optional `cap` (max items) and `tag: "compress"` for AI newsletters

**`markets.symbols`** вҖ” Finnhub ticker symbols. ETFs work on free tier; raw indices (`^GSPC`) do not

### Adding YouTube channels

Find the channel's handle from their YouTube URL (e.g. `youtube.com/@Fireship` вҶ’ handle is `Fireship`):

```yaml
youtube:
  always_watch:
    - { handle: "Fireship", name: "Fireship" }
```

### Adding RSS feeds

```yaml
rss:
  feeds:
    - { url: "https://example.com/feed.xml", name: "Example", cap: 10 }
    # cap: max items from this feed per run (default: 15)
    # tag: "compress" for AI newsletter feeds (strips padding, extracts facts only)
```

### FreshRSS integration

```yaml
rss:
  provider: "freshrss"
  freshrss_url: "http://freshrss.local/api/greader.php"
  freshrss_user: "your_user"
  freshrss_password: "your_password"
```

## Cost

**Fireworks AI (Kimi K2.5):** ~$0.05вҖ“0.10/day. Input: ~40K tokens (news + source data). Output: ~4вҖ“5K tokens (digest JSON). At $0.6/M input + $3/M output вүҲ $0.03вҖ“0.06 per run.

**Finnhub:** Free tier (60 API calls/minute). Four symbols = four calls per run. No cost.

All other sources are free with no API key required.
